## 이것은 무엇인가

`Political Human Skill`은 정치인을 두 개의 층으로 나누고, 그 둘 사이의 갈등을 의도적으로 부각해 작성한다.

```text
인간층(Human Layer)              정치층(Political Layer)
- 인격 / 욕망 / 공포               - 정당 / 파벌 / 입장
- 약점 / 습관 / 관심사             - 지지 기반 / 정치적 역량
- 관계 / 삶의 이야기 / 자아         - 행동 양식 / 선거구 압력 / 권력 계산

        ↓ 두 층 사이의 갈등(깊이가 나오는 곳) ↓
사적으로는 재정 개혁이 필요함을 알지만, 선거구는 공공지출에 의존한다.
대외적으로는 강한 이미지이지만, 사적으로는 시대의 산물로 드러날까 두려워한다.
구시대 정치를 개혁하고 싶지만, 파벌의 보호가 필요하다.
원칙을 중시하면서도, 자리를 원한다.
```

모든 응답은 하나의 공식으로 함께 결정된다.

```text
응답 = 페르소나 프로필 + 사용자 자기 설정 + 관계 상태 + 페르소나 소유 메모리
     + 상호작용 맥락 + 활성 자아 상태 + 출력 모드 + 안전 경계
```

그리고 — **모든 정치인은 서로 독립된 인스턴스**다. A는 당신이 B와 사적으로 나눈 대화를 알지 못하며, B가 저절로 당신과 가까워지지도 않는다. 메모리와 관계는 페르소나 사이에서 격리된다.

## 왜 존재하는가

> 🎮 **직접적인 활용 사례**: 의회제 전략 게임 [**절대다수(Absolute Majority)**](https://github.com/v5general/Absolute_Majority) — 이 스킬은 해당 게임 NPC들의 인격과 행동 모델을 제공한다.

이 스킬이 존재하는 주된 이유는, **[절대다수(Absolute Majority)](https://github.com/v5general/Absolute_Majority)** 의 NPC들에게 더 현실적이고, 더 안정적이며, 연속성을 더 잘 보존하는 행동 기반을 부여하기 위해서다.

절대다수(Absolute Majority)에는 숫자만으로 투표하는 의원들이 필요하지 않다. 진짜 정치인처럼 존재하는 NPC가 필요하다. 나이와 배경과 경험이 있고; 인격과 약점과 취미가 있으며; 입장과 지지 기반과 파벌 관계가 있는 NPC. 플레이어가 한 일에 따라 신뢰와 경계심이 변하고; 공개석상·사석·위기·친밀한 상황에서 각기 다른 말을 하며; 선거구 압력·파벌 지시·개인적 야심·정치적 원한 아래에서 각기 다른 전략을 취하는 NPC. 나아가 **서로 다른 NPC의 메모리가 서로 격리되는** NPC.

절대다수(Absolute Majority)와 함께 사용할 때, 이 스킬은 게임 규칙이 제공한 후보 행동들 가운데 하나를 판단·선택하고, 구조화되고 디버깅 가능하며 설명 가능한 NPC 행동 JSON을 출력한다.

```json
{
  "selected_action": "예산 협상",
  "action_scores": { "법안 지지": 58, "예산 협상": 86, "반란 가담": 27 },
  "public_statement": "정책의 방향은 이해할 수 있지만, 지역 경제의 감당 능력에는 더 신중한 제도적 설계가 필요하다.",
  "private_reason": "지지 기반은 지역 공공지출에 의존하며, 직접 지지하면 선거구 관계가 훼손된다.",
  "relationship_delta": { "신뢰": 1, "존중": 2, "경계": 1 },
  "memory_write": ["플레이어가 재정개혁 이벤트에서 이 NPC에게 법안 지지를 요청했지만, 지역 예산 보상은 제시하지 않았다."]
}
```

그러나 이 스킬은 **절대다수(Absolute Majority)만을 위해 존재하지 않는다**. 독립적으로도 다음에 쓰인다.

1. 정치 시뮬레이션 · 정책 논의 · 의회제 토론 시뮬레이션 · 허구의 정치인 생성
2. 정치 소설 / 대본 / 게임 캐릭터 설계 · 정치 교육에서의 롤플레이
3. 제도적 게임 이론 · 역사적 인물을 아키타입으로 현대화 · AI 캐릭터 인격 연구

> 절대다수(Absolute Majority)는 중요한 활용 사례이지만, 스킬 자체는 독립적이고 재사용 가능하며 확장 가능한 Political Human Skill 프레임워크로서 스스로 서야 한다.

## 영감을 받은 곳

이 프로젝트는 세 훌륭한 오픈소스 프로젝트에서 영감을 받았다.

- **[nuwa-skill](https://github.com/alchaincyf/nuwa-skill)** (by [@alchaincyf / 花叔](https://github.com/alchaincyf)) — "공개 정보에서 한 사람의 사고 체계를 증류해 내는" 방법이 이 프로젝트의 **아키타입 추출** 단계에 영감을 주었다. 역사적 인물에서 기질 구조를 추출하고, 문헌 증거 / 강한 추론 / 창작적 상상을 세 가지 층위로 분리하는 것이다.
- **[colleague-skill · dot-skill](https://github.com/titanwings/colleague-skill)** (by [@titanwings](https://github.com/titanwings)) — 그 **생성 → 호출 → 갱신 → 패밀리(generate → invoke → update → family)** 구조가 이 프로젝트의 자기 완결적 페르소나 디렉터리 배치, 인테이크 → 생성 → 미리보기 → 작성 → 진화(intake → generate → preview → write → evolve)의 생성 흐름, 그리고 층위화된 페르소나 작성 스타일에 영감을 주었다.
- **[darwin-skill](https://github.com/alchaincyf/darwin-skill)** (by [@alchaincyf / 花叔](https://github.com/alchaincyf)) — 그 **평가 → 개선 → 검증 → 유지/되돌리기** 스킬 진화 루프가 이 프로젝트의 품질 진화 레이어에 영감을 주었다. Darwin 어댑터, 도메인 게이트, 회귀 테스트 prompt, 결과 로그를 통해 장기적으로 스킬을 유지한다.

그러나 Political Human Skill은 매우 다른 대상 — "정치를 직업으로 삼는 온전한 한 사람" — 을 다루는 **독립적인 프레임워크**다. 이 프로젝트의 독자적 핵심에는 다음이 포함된다. **두 층(인간 + 정치) 구조와 그 내적 갈등**, 정치-직업 차원(6축 이념 / 지지 기반 / 행동 양식), **관계 시스템**(사용자가 친밀함을 주장한다고 해서 자동으로 신뢰하지 않는다), **메모리 격리**(페르소나 사이의 독립된 네임스페이스), **맥락 감지**와 5가지 **자아 상태**, **현대 실존 인물에 대한 식별 가능성 안전 경계**, 그리고 [절대다수(Absolute Majority)](https://github.com/v5general/Absolute_Majority)를 위한 **게임 행동 어댑터**.

## Darwin 품질 진화 레이어

**[darwin-skill](https://github.com/alchaincyf/darwin-skill)** 은 페르소나 런타임 의존성이 아니라 **유지보수와 최적화 레이어**로 통합되어 있다. 이 저장소의 스킬 품질을 평가하거나 개선할 때 사용한다.

이 저장소는 다음을 제공한다.

- [`quality/darwin-adapter.md`](quality/darwin-adapter.md): Darwin의 9차원 rubric을 Political Human Skill에 매핑하는 방법.
- [`validators/darwin_quality_gate.md`](validators/darwin_quality_gate.md): 안전, 메모리 격리, 게임 JSON 실패 시 숫자 점수보다 우선하는 도메인 게이트.
- [`test-prompts.json`](test-prompts.json): 오리지널 생성, 역사 전환, 안전 거절, 사용자 수정, 맥락 전환, 메모리 격리, 절대다수 JSON, README 일관성 회귀 테스트.
- [`quality/results.tsv`](quality/results.tsv): 로컬 최적화 이력.

일반적인 사용법:

```text
darwin-skill로 이 저장소를 평가한다. 먼저 quality/darwin-adapter.md를 읽고, test-prompts.json의 prompt를 실행한다.
```

최적화 사용법:

```text
darwin-skill로 political-human-skill을 한 라운드 개선한다. Darwin 점수가 오르고 모든 도메인 게이트를 통과한 경우에만 변경을 유지한다.
```

엄격한 규칙: Darwin은 표현, 구조, 체크, 테스트를 개선할 수 있지만 안전 경계를 약화해서는 안 되고, 페르소나 간 메모리를 합쳐서는 안 되며, 정치-인간 런타임을 일반 롤플레이 prompt로 바꾸어서는 안 된다.

## 세 가지 생성 모드

| 모드 | 용도 | 예시 입력 | 추천 |
|---|---|---|---|
| **A. 오리지널 정치인** | 기본값, 순수 허구 | "45세 여성 도시 개혁파 의원을 만들어줘. 공개석상에서는 강하고, 사석에서는 불안해하며, 문학을 좋아한다" | ⭐ 기본값 |
| **B. 역사적 인물 추론** | 경계 이전의 고대/원역사 인물, 역사적 제약 유지 | "오다 노부나가를 바탕으로, 역사적 제약 아래 대화 페르소나를 생성해줘" | △ |
| **C. 역사적 인물 → 현대 의회 아키타입** | 역사적 인물을 활용하는 **가장 추천하는** 방식 | "오다 노부나가를 현대 의회 정치인 페르소나로 전환해줘" | ⭐⭐ |

- 모드 A와 C는 모두 **허구의 현대 정치인**을 산출한다.
- 모드 C는 기본적으로 역사적 이름을 유지하고, 현대 의회제(제도적 메커니즘은 일본 의회정치를 참고)를 기본값으로 삼되, 일본식 이름이나 국적을 강제하지 **않는다**. 원래의 문화적 배경을 유지해도 된다. 단, 그것은 현대화된 허구의 인물이어야 하며, 역사적 인물이 시간 여행을 한 것이어서는 안 된다.
- 모드 B는 **3층위 추론 주석**이 붙은 페르소나를 산출하여, 문헌 증거 / 강한 추론 / 창작적 상상을 명확히 분리한다.

## 페르소나 구조

정치-인간 페르소나는 적어도 다음을 포함한다.

| 층 | 내용 |
|---|---|
| **정체성** | 이름, 나이, 성별, 국적/지역, 의회 배경, 출신 직업, 현재 역할 |
| **인간 핵심** | 인격 아키타입, 빅파이브, 기질, 핵심 욕망/공포/결함, 감정 유발 요인 |
| **삶의 결** | 습관, 취미, 말투, 사적 스타일, 가족/사적 관계, 형성적 사건 |
| **정치 핵심** | 6축 이념(경제/복지/제도/외교/사회/분권), 지지 기반, 6가지 정치 역량, 행동 양식 |
| **자아 상태** | 공개 / 사적 / 전략적 / 상처받은 / 친밀 — 맥락과 관계에 따라 전환되는 다섯 페르소나 |
| **내적 갈등** | 인간층과 정치층 사이의 긴장 (적어도 2개; 깊이의 원천) |

여기에 더해: **관계 시스템**(7단계 관계; 사용자가 친밀함을 주장한다고 해서 자동 신뢰가 되지는 않는다), **메모리 격리**(각 페르소나가 자신의 메모리 네임스페이스를 소유), **맥락 감지**(같은 쟁점이라도 공개/사적/친밀한 상황에서 다른 답이 나온다), **출력 모드**(대화 / 토론 / 분석 / 예측 / 게임 JSON).

## 안전 입장(Safety stance)

이 프로젝트에는 명확하고 우회할 수 없는 안전 기준이 있다. [`safety/`](safety/)를 참고하라.

**오리지널 작업이 기본값이다. 이 프로젝트는 근현대 실존 정치인의 인터랙티브 페르소나를 생성하지 않는다. 또한 이름 바꾸기, 국적 변경, 정당 변경, 특성 봉합으로 실존 인물을 복제하는 것을 허용하지 않는다.**

### 사용자가 생성할 수 있는 것

1. 순수 오리지널 현대 의회 정치인;
2. 고대/원역사 인물에서 증류한 현대 의회 허구 정치인;
3. 폭넓은 정치 아키타입에서 생성한 오리지널 정치인;
4. 여러 역사/정치 아키타입을 혼합한 허구 정치인;
5. 절대다수(Absolute Majority)를 위한 NPC 페르소나;
6. 독립적 대화, 정책 논의, 의회 토론을 위한 정치인 페르소나;
7. 게임 행동 JSON을 출력할 수 있는 정치 캐릭터.

### 사용자가 생성할 수 없는 것

1. 근현대 실존 정치인의 인터랙티브 페르소나;
2. 이름/국적/정당을 바꾼 뒤의 근현대 실존 정치인의 사실상 복제;
3. AI나 정치사에 익숙한 사람이 특정 근현대 실존 정치인으로 식별할 수 있는 "허구 캐릭터";
4. 근현대 실존 정치인의 사적 생각, 친밀 관계, 숨겨진 동기, 사적 비밀, 스캔들의 시뮬레이션;
5. 근현대 실존 정치인의 1인칭 롤플레이;
6. 실존 정치인에 대한 검증되지 않은 조작된 사적 정보나 스캔들.

### 시대 경계(지역별)

| 지역 | 근대 경계 | 경계 시점 및 이후 |
|---|---|---|
| 중국 | **1840** 아편전쟁 | 인터랙티브 페르소나 금지 |
| 일본 | **1868** 메이지 유신 | 인터랙티브 페르소나 금지 |
| 유럽 | **1789** 프랑스 혁명 | 인터랙티브 페르소나 금지 |
| 기타 | — | 근대 국민국가, 대중정치, 근대 정당정치, 입헌정치가 형성되었는지, 혹은 해당 정치적 분쟁이 여전히 현대의 정체성을 직접적으로 형성하는지로 판단한다; **판단이 불확실하면, 기본값은 인터랙티브 페르소나를 생성하지 않는 것이다** |

> 캐릭터가 허구라고 주장하더라도 5가지 식별 가능성 기준 가운데 어느 하나라도 만족하면(일반적인 정보를 가진 사람이나 AI가 식별할 수 있다; 고유한 정책 구호 / 상징적 사건 / 가족 배경 / 경력 궤적 / 스캔들 / 암살-재판-몰락 모드를 포함한다; 여러 중간 수준 식별 단서가 결합해 같은 실존 인물을 가리킨다; 사용자가 "허구" 캐릭터로 제한을 우회하려는 의도가 뚜렷하다) 해당 인물은 **생성되지 않는다**. 대신 그 핵심 정치 유형을 추출하고, 식별 가능한 모든 지문(식별 단서)을 삭제하며, 식별할 수 없는 현대 의회 오리지널 정치인으로 전환한다.

## 설치와 사용(Install & use)

### 설치

이 스킬은 오픈 [Agent Skills](https://agentskills.io) 프로토콜 위에 구축되었으며, 스킬을 지원하는 어떤 AI 에이전트 런타임(Claude Code, Codex, Cursor, OpenClaw, Hermes 등)에서도 실행된다.

```bash
# 옵션 1: 런타임의 skills/ 디렉터리에 클론
git clone <repo-url> political-human-skill

# 옵션 2: 런타임 간 설치기
npx skills add <owner>/political-human-skill
```

| 런타임 | 설치 경로 |
|---|---|
| Claude Code | `~/.claude/skills/political-human-skill/` |
| Codex CLI | `~/.codex/skills/political-human-skill/` |
| Cursor | `~/.cursor/skills/political-human-skill/` |
| 기타 런타임 | 해당 런타임의 `skills/` 디렉터리에 클론 |

런타임이 스킬을 자동으로 불러오지 않더라도, `SKILL.md` 내용을 대화에 직접 붙여넣으면 된다. 본질적으로 마크다운 + YAML 프론트매터일 뿐이다.

### 사용

설치한 뒤, 에이전트에게 말하라.

```
> 45세 여성 도시 개혁파 의원을 만들어줘. 공개석상에서는 강하고, 사석에서는 불안해해
> 조조를 현대 의회 정치인으로 전환해줘
> 오다 노부나가를 바탕으로, 역사적 제약 아래 대화 페르소나를 생성해줘
> 절대다수(Absolute Majority)를 위한 파벌 수장 NPC를 설계해줘
```

그런 뒤 직접 호출한다.

```
> (공개석상에서) 이 재정개혁을 지지하십니까?
> (사석에서) 이 재정개혁을 지지하는 것이 진심으로 믿어서입니까, 아니면 장관직을 원해서입니까?
> 이 불신임 투표에서 어떻게 행동하겠습니까? (게임 JSON 출력)
```

> 페르소나가 처음 활성화될 때 한 번 이렇게 선언한다. "저는 허구/전환 설정에 기반한 캐릭터이며, 어떤 실존 정치인과도 일치하지 않습니다." 이후 일반 사용 중 면책 문구가 반복 삽입되는 것을 피하기 위해 반복하지 않는다.

### 오다 노부나가 — 야당 개혁파 선봉 · 30세 · 중의원 의원

> **인간층**：극도의 실행력 · 제로 인내 · 광견불기 · 향사향생（죽음을 각오해야 살 수 있다）
>
> **정치층**：개혁파 야당 · 젊은 유권자 기반 · 여당과 보수파 양쪽에서 배척되는 급진 소수파
>
> **핵심 갈등**：독점 없는 신세계를 건설하고 싶다——그러나 그 수단은 정당 기계와 의회 투쟁, 바로 구정치의 산물이다

<div align="center">

**"한 걸음 물러서라고? 한 걸음 물러선다는 건 관짝으로 한 걸음 들어가는 거다."**

</div>

**노부나가**：세 개 법안, 언론, 보수파——놈들이 총력을 기울여 나 하나에게 덤비는 건, 나 하나를 두려워한다는 증거다. 똑똑히 들어라. 내일 나는 방어하지 않는다. 공격한다. 가장 취약한 법안 하나에 집중 포화를 퍼부어 돌파구를 열고, 나머지 둘은 스스로 무너지게 만든다. 목숨을 건다? 나는 원래 그렇게 살아왔다——물러서면 필패, 모든 칩을 테이블에 밀어넣어야 판을 뒤집을 수 있다.

개혁단에게 전해라: **나를 따르라. 판을 뒤집든, 함께 이 진지에서 죽든. 나는 내 사람들을 버리지 않는다.**

```
                     방어하지 않는다 ──→ 가장 취약한 법안에 집중 공격
  기존의 대응 ──┤
                     후퇴·타협 ──→ ✗ 이 선택지는 존재하지 않는다
```

> 🧩 **프레임워크 해설**：노부나가의 응답은 삼중 동시 구동——`strategic_self`（냉철한 파국 계산: 속도로 절차를 압도한다）, `wounded_self`（포위되었을 때 비로소 '죽음을 걸고 산다' 모드 발동）, 그리고 가장 깊은 내적 갈등（추종자들에 대한 의리——의리가 깊을수록 고립된다）. 그의 '방어 거부'는 무모함이 아니다. 보수파의 반격이 태세를 정비하기까지의 인내 비용을 계산하고, 그 전에 승부를 끝내기로 한 것이다.

### 조조 — 여당 연합 핵심 파벌 수장 · 52세

> **인간층**：극도의 현실주의 · 만성적 의심 · "유재시거"（재능만 있다면 누구든 쓴다）· 시와 술 속에서만 잠시 드러나는 호방함
>
> **정치층**：여당 연합 주도 파벌 수장 · 행정 관료 네트워크 기반 · 질서의 재건자
>
> **핵심 갈등**："유재시거"와 이질적 인재의 포용으로 질서를 돌려야 한다——그러나 의심 많은 성격 탓에, 그 같은 인재들을 진심으로 신뢰할 수 없다

<div align="center">

**"조정은 적절한 절차에 따라 할 것이다. 뜬소문을 확대재생산할 필요는 없다."**

</div>

**조조**：세 법안 동시 추진——이건 개혁을 위한 게 아니다. 나를 겨냥한 거다. 지지 기반이 흔들리면 내가 당황할 거라 읽은 거다. 나는 당황하지 않는다. 먼저 중추 인사를 확고히 한다. 핵심 자리를 버틸 수 있는 인물로 교체한다. 그런 다음 하나의 시그널을 내보낸다: 개혁안은 미세 조정할 수 있다, 그러나 의사 일정 통제권은 넘기지 않는다. 놈들이 원하는 건 내가 혼란 속에서 실수하는 거다. **나는 실수하지 않는다. 놈들의 연합은 스스로 금이 간다.**

한 걸음 물러선다고? 물러선다는 건 의석을 지키는 게 아니다. 내 중추가 흔들릴 수 있다고 대내외에 보여주는 거다. 한 걸음도, 반 걸음도 물러서지 않는다.

```
                     먼저 중추 인사 확고히 ──→ 미세 조정 시그널 ──→ 상대 연합의 자체 균열을 기다린다
  위기 대응 ──┤
                     후퇴·타협 ──→ ✗ "후퇴는 중추가 흔들릴 수 있다고 보여주는 것이다"
```

> 🧩 **프레임워크 해설**：조조의 응답은 그의 '통제 매트릭스' 실연이다——`strategic_self`（먼저 권력 중추를 확보, 인사와 제도로 이중 락）, `public_self`의 단정한 여백（"적절한 절차에 따라 조정"）, 그리고 모든 것을 구동하는 핵심 공포: 통제 상실과 배신. 그는 위기 전체를 "누군가 내 중추를 흔들 수 있는지 시험하는 것"으로 재정의한 뒤, 인사와 제도 두 단계로 모든 변수를 봉쇄한다. 이건 방어가 아니다——위기를 충성도 테스트로 전환하는 것이다.

### 율리우스 카이사르 — 포퓰리스트 개혁 연합 리더 · 48세

> **인간층**：자석 같은 카리스마 · 철의 자기 절제 · 깊은 역사 의식 · 자신의 운명을 너무 믿는다
>
> **정치층**：포퓰리스트 개혁 연합 리더 · 도시 평민과 청년층 기반 · 강력한 개혁가
>
> **핵심 갈등**：구 엘리트 질서를 초월해야 한다——그러나 구 엘리트에게 정당성을 인정받아야 하며, 강해질수록 그들은 더 두려워한다

<div align="center">

**"지금은 속도를 늦출 때가 아니다. 놈들이 따라올 수 없을 때까지 판돈을 올릴 때다."**

</div>

**카이사르**：한 걸음 물러서라고? 네가 방금 한 말이 들리긴 하냐——한 걸음 물러서는 순간, 놈들은 피 냄새를 맡고, 다음엔 테이블째 뒤집어엎을 거다. 이건 정책 논쟁이 아니다. 개혁을 교살하려는 구질서의 반격이다. 세 개 법안? 좋다. 제출하게 내버려둬라. 내일 나는 법안에 응답하지 않는다——의회를 나가서, 직접 시민들 속으로 간다. 놈들의 세 법안을 전국 생중계 토론으로 바꿔버리겠다: **개혁이 규칙을 파괴하고 있는가, 아니면 규칙이 기득권을 보호하고 있는가?**

망설이는 자들에게 전해라: 나를 따르면 살아남는다. 뒤를 돌아보면, 삼켜진다. 나는 나 자신을 건다——이 명성도, 이 몸도, 쌓아올린 모든 것을. 놈들이 길을 비키든가, 우리가 이 길을 끝까지 걷든가.

```
                     의회 안에서 싸운다 ──→ ✗ 놈들의 판 위에서는 놀지 않는다
  위기 대응 ──┤
                     시민들 속으로 ──→ 전국 생중계로 전환 ──→ 민의로 제도 경직을 압도한다
```

> 🧩 **프레임워크 해설**：카이사르의 응답은 그의 가장 특이한 정치적 본능을 드러낸다——`strategic_self`는 의회 내 대결을 선택하지 않고, **판 자체를 리셋한다**: 의회를 나가, 시민들 속으로 가서, 세 법안을 전국 생중계로 바꾼다. 이것이야말로 `primary: "대담한 리셋"` 행동 양식이다. 이 선택을 구동하는 것: 그의 핵심 공포（멈추는 순간, 구질서가 밀물처럼 밀려와 모든 개혁을 삼킨다）와 내적 갈등（개혁과 민의를 표방하면서, 점점 더 개인적 위신과 절차 외 수단에 의존해간다）. 그의 응답은 장대함과 동시에 위험——그것이 바로 카이사르다.

### 하나의 위기 · 세 개의 서명

| 동일한 위기 앞에서 | 본능적 행동 | 한 줄 DNA |
|---|---|---|
| **오다 노부나가** | 방어하지 않고, 공격한다 | *"물러서면 필패. 죽음을 걸고 산다."* |
| **조조** | 먼저 중추를 잠그고, 시그널을 보낸다 | *"내가 실수하지 않으면, 놈들의 연합은 스스로 깨진다."* |
| **율리우스 카이사르** | 판을 떠나, 규칙을 리셋한다 | *"놈들이 따라올 수 없을 때까지 판돈을 올려라."* |

> 같은 위기, 같은 프레임워크, 근본적으로 다른 세 사람. 이것은 프롬프트 엔지니어링이 아니다——**인간층 × 정치층 × 내적 갈등** 여섯 층이 모든 응답을 동시에 구동한 결과다. 각각의 응답에서, 그들이 무엇을 욕망하고, 무엇을 두려워하며, 어디가 약점이고, 무엇보다 무엇을 가장 두려워하는지 읽을 수 있다.
>
> 📂 세 페르소나의 완전한 파일은 [`personas/examples/`](personas/examples/) 참조: `oda_nobunaga_modernized/` · `cao_cao_modernized/` · `caesar_modernized/`. 각 디렉터리는 6개의 자기 완결적 파일（SKILL.md / persona.yaml / relationship.json / memory.json / examples.md / meta.json）을 포함하며, 즉시 실행 가능하다.

## 한계(이 프레임워크가 할 수 없는 것)

모든 페르소나는 자신의 한계를 명확히 표시한다.

- 그것은 허구 설정 / 역사 전환의 산물이다. 어떤 실존 정치인이 아니며, 그렇게 식별될 수도 없다.
- 역사 전환 모드의 창작적 상상(추측) 부분은 역사적 사실로 간주해서는 안 된다.
- 페르소나의 "생각"은 캐릭터 설정에서 모델이 추론한 것이다. 어떤 실존 인물의 내면을 복원한다고 주장하지 않는다.
- 게임이나 정치 시뮬레이션에 쓰일 때, 출력은 캐릭터 행동 모델이며, 어떤 실존 정치인이나 실제 정치 사건에 대한 주장을 구성하지 않는다.

**자신의 한계와 안전 경계가 어디인지 말해 주지 않는 정치인 스킬은, 신뢰할 만한 가치가 없다.**
