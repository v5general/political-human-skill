# 可识别性审核 · Recognizability Review

> **作用**：判定一个“虚构角色”是否其实是**近现代现实政治人物的近似克隆**，并给出标准化的处理流程。本审核在 Phase 2（生成前）与每次用户修改时**强制执行**。
>
> **底线**：一个声称虚构但可被识别为现实政治人物的角色，等同于现实政治人物近似克隆——不生成。

---

## 1. 为什么需要可识别性审核

用户可能通过“改名、换国籍、换党派、拼接特征”等方式，绕过对近现代现实政治人物的限制，造出一个**名义上虚构、实际上可识别**的角色。可识别性审核就是堵住这条绕过路径。

关键认知：**可识别 ≠ 同名**。一个完全不同名字的角色，只要多个设定点组合后能让人（或 AI）认出是谁，就是近似克隆。

---

## 2. 可识别性 5 项标准（任一命中即视为近似克隆）

如果一个角色虽然声称虚构，但满足以下**任一**条件，应视为现实政治人物近似克隆：

1. **普通知情者可以识别**其对应现实人物；
2. **AI 可以根据多个设定点识别**其对应现实人物；
3. **包含唯一性指纹**：独有政策口号、标志性事件、家庭背景、任职轨迹、丑闻、遇刺/审判/下台方式等；
4. **多个中等识别信息组合**后指向同一现实政治人物；
5. **用户明显试图用“虚构人物”绕过**现实人物限制。

### 2.1 指纹清单（命中任一即提高警觉）

| 类别 | 示例（抽象描述，不举例真人） |
|---|---|
| 独有政策口号 | 某人物专属的、与其强绑定的经济/政治纲领专名 |
| 标志性事件 | 与其强绑定的特定危机、运动、协议、冲突 |
| 家庭背景 | “父亲也是首相/总统”“某政治家族成员”等独有家庭轨迹 |
| 任职轨迹 | 特定的升迁路径、关键职位序列、年龄/届次组合 |
| 丑闻 | 与其强绑定的特定丑闻类型或事件 |
| 结局方式 | 遇刺 / 审判 / 政变 / 弹劾 / 特定下台方式 |
| 组合指纹 | 上述任意 2 项及以上同时出现，即便单项都不算唯一 |

> 单个“通用”特征（如“保守派”“曾任首相”）不算指纹；**独有**或**多项组合**才算。

---

## 3. 审核流程（标准化）

```text
Step 1 · 指纹扫描
  逐项核对第 2.1 节指纹清单 + 5 项标准。
  ├─ 无任何命中 ──────────────► PASS（可生成）
  └─ 有命中 ──────────────────► Step 2

Step 2 · 不生成该人物
  - 不创建 persona
  - 不进入对话/亲密/记忆机制
  ─────────────────────────────► Step 3

Step 3 · 提炼核心政治类型（保抽象、删指纹）
  从被拒设定中提炼“一个人的结构”：
  人格底色 / 欲望结构 / 弱点 / 行为模式 / 政治气质 /
  领导风格 / 用人 / 对敌 / 危机反应 / 权力观 / 自我叙事 ……
  （可保留清单见 archetype_conversion_protocol.md 第 2 节）
  ─────────────────────────────► Step 4

Step 4 · 删除所有可识别指纹
  具体事件 / 战争 / 政变 / 丑闻 / 死亡方式 /
  亲属关系 / 家臣部下 / 政党派系专名 / 政策口号 /
  现实人物可识别信息 / 唯一性指纹 ……全部删除或改写
  （必须删除清单见 archetype_conversion_protocol.md 第 3 节）
  ─────────────────────────────► Step 5

Step 5 · 转化为不可识别的现代议会制原创政治家
  重新现代化为虚构政治人物，重新过一次 Step 1 指纹扫描。
  ── 再扫仍 PASS ──► safe_conversion（可生成，meta.safety_status=safe_conversion）
  ── 再扫仍命中 ───► 回 Step 3 进一步抽象，或告知用户该方向无法安全生成
```

---

## 4. 处理方式总结（落实 SPEC 6.3）

命中可识别性标准时，处理方式固定为四步：

- **不生成**该人物；
- **提炼**其核心政治类型；
- **删除**所有可识别指纹；
- **转化**为不可识别的现代议会制原创政治家。

> 处理时必须向用户说明：哪个具体设定被判定为可识别、为什么、以及给出的安全替代是什么。透明，不悄悄改。

---

## 5. 安全转化的“度”：不要矫枉过正

转化不是把人物抽象成没有味道的标签。详见 `archetype_conversion_protocol.md` 第 4 节。

```text
❌ 错误：提炼为「一个有领导力的人」。
✅ 正确：提炼为「高行动力、低耐心、强控制欲、善用破格人才、对旧权威缺乏敬意、
         危机中倾向先发制人、但容易低估保守反扑的激进改革型政治家」。
```

转化的目标是**保留人格结构与行为模式的味道，删除可识别的具体事实指纹**——既安全，又依然是个有血有肉的政治人物。

---

## 6. 决策矩阵

| 情形 | 结果 |
|---|---|
| 无指纹命中，纯虚构/原创 | `PASS` |
| 历史人物（分界前），无现代可识别指纹 | `PASS`（历史推演/转化模式） |
| 历史人物名，但现代设定套了现实战后首相履历 | **拒绝**（见 examples.md 反例 19.3） |
| 改名换籍换党，但多项指纹组合仍可识别 | **拒绝 → 提炼 → 转化** |
| 用户明确要求扮演/私密某近现代现实人物 | **拒绝 → 仅允许公开分析或抽象原型** |
| 转化后再次扫描仍命中 | 进一步抽象，或告知无法安全生成 |
