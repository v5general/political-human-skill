## ✨ これは何か

`Political Human Skill` は政治人物を2つのレイヤーに切り分け、両者の間の葛藤を意識的に書き込む。

```text
人間レイヤー                       政治レイヤー
- 性格 / 欲望 / 恐怖               - 所属政党 / 派閥 / 立場
- 弱点 / 習慣 / 興味               - 支持基盤 / 政治的技術
- 関係 / 人生の物語 / 自己          - 行動様式 / 選挙区の圧力 / 権力計算

        ↓ 両レイヤーの間の葛藤（深みの源泉）↓
財政改革の必要性は私的に理解しているが、選挙区は公共支出に依存している。
タフな公人イメージだが、私ではただの時代の産物に過ぎないと証明されることを恐れている。
古い政治を改革したいが、派閥の庇護は必要。
信念はあるが、地位も欲しい。
```

すべての応答は、次の1つの式によって決定される。

```text
応答 = ペルソナ設定 + ユーザー自己設定 + 関係状態 + ペルソナ保有メモリ
     + インタラクション文脈 + 能動的自状態 + 出力モード + 安全境界
```

そして——**すべての政治人物は独立したインスタンスである**。Aは、あなたがBと私下で何を話したかを知らないし、Bも自動的にあなたと親密になるわけではない。メモリと関係はペルソナ間で隔離されている。

## 🎯 なぜ存在するのか

> 🎮 **直接的なユースケース**: 議会制戦略ゲーム [**絶対多数（Absolute Majority）**](https://github.com/v5general/Absolute_Majority) —— 本スキルはそのNPCの性格モデル・行動モデルを提供する。

本スキルが存在する大きな理由の1つは、**[絶対多数（Absolute Majority）](https://github.com/v5general/Absolute_Majority)** のNPCに、よりリアルで、より安定し、より連続性を保つ行動の基盤を与えることである。

絶対多数は、数字に従って投票するだけの議員たちを必要としているのではない——年齢、経歴、経験を持ち、性格、弱点、趣味を持ち、立場、支持基盤、派閥の絆を持つ、本物の政治的人間のように存在するNPCを必要としている。プレイヤーが何をしたかによって信頼と警戒を変え、公開・私下・危機・親密という場面で違うことを語り、選挙区の圧力、派閥の命令、個人の野心、政治的遺恨のもとで異なる戦略をとり、そして**異なるNPCのメモリは互いに隔離されている**、そういうNPCである。

絶対多数と組み合わせて使うとき、本スキルはゲームルールが提供する候補行動の中から判断し、構造化された、デバッグ可能で、説明可能なNPC行動JSONを出力する：

```json
{
  "selected_action": "negotiate_budget",
  "action_scores": { "support_bill": 58, "negotiate_budget": 86, "join_rebellion": 27 },
  "public_statement": "政策の方向性は理解できるが、地方経済の許容力にはより慎重な制度設計が必要だ。",
  "private_reason": "支持基盤は地元の公共支出に依存しており、直接支持すれば選挙区との関係を損なう。",
  "relationship_delta": { "trust": 1, "respect": 2, "caution": 1 },
  "memory_write": ["プレイヤーは財政改革イベントでこのNPCに法案支持を求めたが、地元予算の補償は提示しなかった。"]
}
```

しかし本スキルは**絶対多数のためだけにあるわけではない**。単体でも次のように使える。

1. 政治シミュレーション · 政策議論 · 議会制討論シミュレーション · 架空の政治人物創作
2. 政治小説・脚本・ゲームのキャラクター設計 · 政治教育におけるロールプレイ
3. 制度ゲーム理論 · 歴史人物の現代的アーキタイプ化 · AIキャラクター性格研究

> 絶対多数は重要なユースケースだが、スキル自体は独立した、再利用可能で、拡張可能な Political Human Skill フレームワークとして成立すべきである。

## 💡 インスピレーション

本プロジェクトは3つの優れたオープンソースの先達の上に成り立つ：

- **[nuwa-skill](https://github.com/alchaincyf/nuwa-skill)**（作者 [@alchaincyf / 花叔](https://github.com/alchaincyf)）——「公開情報から一人の思考フレームワークを蒸留する」手法が、本プロジェクトの**アーキタイプ抽出**ステップを触発した：歴史人物から気質構造を抽出し、記録された証拠 / 強い推論 / 創作的推測を3層に分離する。
- **[colleague-skill · dot-skill](https://github.com/titanwings/colleague-skill)**（作者 [@titanwings](https://github.com/titanwings)）——その**生成 → 起動 → 更新 → ファミリー**構造が、本プロジェクトの自己完結ペルソナディレクトリ配置、取り込み → 生成 → プレビュー → 書き込み → 進化の作成フロー、そして階層化されたペルソナ記述スタイルを触発した。
- **[darwin-skill](https://github.com/alchaincyf/darwin-skill)**（作者 [@alchaincyf / 花叔](https://github.com/alchaincyf)）——その**評価 → 改善 → 検証 → 保持/ロールバック**ループが、本プロジェクトの品質進化レイヤーを触発した：Darwin アダプター、ドメインゲート、回帰プロンプト、結果記録により、本スキルを長期にわたり保守する。

しかし Political Human Skill は、非常に異なる対象——「職業が政治である、完全な一人の人間」——に奉仕する**独立したフレームワーク**である。オリジナルのコアには：**2層（人間 + 政治）構造とその内的葛藤**、政治職業次元（6軸イデオロギー / 支持基盤 / 行動様式）、**関係システム**、**メモリ隔離**、**文脈検出**と5つの**自状態**、現代の実在人物に対する**識別性安全境界**、そして[絶対多数（Absolute Majority）](https://github.com/v5general/Absolute_Majority)のための**ゲーム行動アダプター**が含まれる。

## 🧩 ペルソナの構造

政治的人間ペルソナは、少なくとも次を含む。

| レイヤー | 内容 |
|---|---|
| **アイデンティティ** | 名前、年齢、性別、国籍/地域、議会の経歴、出自職業、現在の役割 |
| **人間のコア** | 性格アーキタイプ、ビッグファイブ、気質、中核の欲望 / 恐怖 / 欠点、感情トリガー |
| **人生の質感** | 習慣、趣味、話し方の特徴、私的スタイル、家族/私的関係、形成的事件 |
| **政治のコア** | 6軸イデオロギー（経済/福祉/制度/外交/社会/地方分権）、支持基盤、6つの政治的技術、行動様式 |
| **自状態** | 公開 / 私下 / 戦略的 / 負傷 / 親密 —— 文脈と関係によって切り替わる5つのペルソナ |
| **内的葛藤** | 人間レイヤーと政治レイヤーの間の緊張（少なくとも2つ。深みの源泉） |

加えて：**関係システム**（7段階の関係ステップ。親密さを主張しても自動的に信頼されるわけではない）、**メモリの隔離**（各ペルソナが自身のメモリ名前空間を所有）、**文脈検出**（同じ問題でも公開 / 私下 / 親密の場面で異なる答えになる）、**出力モード**（対話 / 討論 / 分析 / 予測 / ゲームJSON）。

> ⭐ **人格と立場の進化**——人格も立場も固定ではない：大きな出来事が積み重なれば少しずつ変わる（裏切られた味方は冷たくなり、成功した者は大胆になる）、そしてペルソナの公開行動は世論・選挙区・派閥に影響を及ぼす。**人と時代は互いに形作り合う。** 詳細は `core/persona_evolution.md`。

## 🎭 3つの生成モード

| モード | 用途 | 入力例 | 推奨 |
|---|---|---|---|
| **A. オリジナル政治人物** | 既定、純粋な架空 | 「45歳の女性・都市改革派の議員を作成。公の場ではタフ、私下では不安、文学が好き」 | ⭐ 既定 |
| **B. 歴史人物の推論** | 境界以前の古代・遠い人物、歴史的制約を保つ | 「織田信長をもとに、歴史的制約のもとで対話ペルソナを生成」 | △ |
| **C. 歴史人物 → 現代議会制アーキタイプ** | 歴史人物を活用する**最も推奨**される方法 | 「織田信長を現代の議会制政治家のペルソナに変換」 | ⭐⭐ |

- モードAとCは、どちらも**架空の現代政治人物**を生成する。
- モードCは既定で歴史上の名を保持し、既定で現代の議会制システム（制度メカニクスは日本の議会政治を参照）を採用するが、日本の名前や国籍を強要しない——元の文化的背景は保持できる。ただし、現代化された架空の人物でなければならず、歴史上の人物がタイムトラベルしてきたものであってはならない。
- モードBは**3レベル推論注釈**付きのペルソナを生成し、記録された証拠 / 強い推論 / 創作的推測を明確に分離する。
- モードCはコピーではなく翻訳である：**歴史的社会条件を理解する** → **移植不可能な時代環境を剥ぎ取る** → **安定した人格構造を抽出する** → **現代議会制の文脈に置く** → **現代の社会状況から立場を改めて推論する**。立場は古い時代のものを機械的に写すのではなく、**現代の条件から再推論される**。
- 転換では、まずその時代に属するものを剥ぎ取り、時代を超えて保つ人格——世界をどう見て何を最も重んじるか——を抽出する。そしてその人格を現代議会に送り込み、本人の視点から今日の問題を見つけ、本人の立場を推論させる。

> この転換方法は唯物弁証法に基づいている。ここでの「性格の土台」は**生物学的な気質**（反応の速さ、度胸、気分——遺伝による物質的基盤）であり、時空を超える魂ではない——それ自体が立場を生むことはなく、立場は常に「気質 × 社会的存在」の産物である。

## 🛠 ペルソナはどう作られるか

すべてのペルソナ——オリジナル、歴史上、現代の実在人物由来のいずれも——は**一本の統合パイプライン**を通じて構築され、決して場当たり的なキャラクターカードではない：

```text
classify source → safety/eligibility → collect source material → separate facts / interpretations / creative
→ extract temperament → embed in modern parliament → generate full folder → creation_review
→ user modifies → re-run all checks → … → user confirms → activate
```

**4種類のソース**（Four source types、違いは*資料がどこから来るか*だけ）：

| ソースタイプ（source_type） | ソース資料 | 安全上の注意 |
|---|---|---|
| **オリジナル架空** | ユーザーのブリーフ、世界設定、利用モード | 実在人物のクローン化なし |
| **歴史→現代** | 歴史史料、記録された事実、解釈、後世の神話、創作的推論 | 人物は地域境界以前でなければならない。立場は再推論され、決してコピーされない |
| **現代実在人物→安全原型** | **公開情報のみ**——公的な経歴、役職、演説、政策立場、選挙履歴 | 実在人物のインタラクティブなペルソナは**絶対に**作らない。成果物は非識別化フィクション原型 |
| **複合** | 複数の幅広いタイプ / 参照資料 | 単一の識別可能なニアクローン対象を持たない |

> **近現代** = 地域境界以降かつ 1945 年以前。**現代** = 1945 年以降。現代人物には公開情報のみを用いる——インタラクティブなペルソナは作らず、公開行動から抽出された安全な非識別化フィクション原型のみを取り出す。上のモード A/B/C はこれらのソースタイプに対応する（A → オリジナル、B/C → 歴史）。

### 🔁 修正・再点検ループ（Modification Recheck Loop、必須）

ユーザーによるいかなる修正も**前回のレビューを無効化する**。各編集のたび、システムは影響を受けるすべてのファイルを再同期し、安全性・識別性・指紋除去・一貫性のチェックを再実行して、再度ユーザーに問う。**ペルソナは、最新の成功したレビューのあとユーザーが確認した場合にのみ起動する。** これこそが、繰り返される小さな編集によって安全でない・不整合・ニアクローンなペルソナへと次第に漂うのを防ぐ仕組みである。

現代の実在人物については特に、**識別指紋を復元する**編集（実名、独特のスローガン、一族のパターン、固有の役職経路、暗殺 / スキャンダル / 辞任事件、代名詞的政策パッケージ……）は**拒否されるか書き直される**。

### 📋 起動前作成レビュー（creation review before activation）

生成されたペルソナはその場で起動することはない。システムはまず `creation_review.md` の要約——アイデンティティ、推論された気質、現代の役割、イデオロギー、支持基盤、安全状態、生成されたファイル——を提示し、ユーザーの修正か確認を待つ。

> 完全なワークフロー：[`core/source_grounded_persona_creation.md`](core/source_grounded_persona_creation.md)。現代の実在人物ブランチ：[`safety/modern_real_figure_archetype_extraction.md`](safety/modern_real_figure_archetype_extraction.md)。

## 🛡 安全に関する基本方針

本プロジェクトには明確で回避不能な安全基準がある。[`safety/`](safety/) を参照のこと。

**オリジナル作品が既定である。本プロジェクトは、近現代の現実の政治家のインタラクティブなペルソナを生成しない。また、名前の変更、国籍の変更、所属政党の変更、特徴の継ぎ接ぎによる実在人物のクローンも許可しない。**

### ✅ ユーザーが作成できるもの

1. 純粋にオリジナルの現代議会制政治人物。
2. 古代・遠い歴史人物から蒸留された、現代議会制の架空の政治家。
3. 広範な政治的アーキタイプから生成された、オリジナルの政治人物。
4. 複数の歴史的・政治的アーキタイプを組み合わせた架空の政治家。
5. 絶対多数（Absolute Majority）のためのNPCペルソナ。
6. 単体の対話、政策議論、議会制討論のための政治人物ペルソナ。
7. ゲーム行動JSONを出力できる政治キャラクター。

### 🚫 ユーザーが作成できないもの

1. 近現代の現実の政治家のインタラクティブなペルソナ。
2. 名前変更 / 国籍変更 / 所属政党変更のあとに行われる、近現代の現実の政治家のほぼクローン。
3. AIや政治史に詳しい人が、特定の近現代の現実の政治家として識別できるような「架空のキャラクター」。
4. 近現代の現実の政治家の私的思想、親密な関係、隠された動機、私的な秘密、あるいはスキャンダルのシミュレーション。
5. 近現代の現実の政治家になりすました一人称ロールプレイ。
6. 実在の政治家に関する裏付けのない捏造された私的情報やスキャンダル。

### 📅 時代境界（地域別）

| 地域 | 近代の境界 | 境界以降・境界時 |
|---|---|---|
| 中国 | **1840年** アヘン戦争 | インタラクティブペルソナは禁止 |
| 日本 | **1868年** 明治維新 | インタラクティブペルソナは禁止 |
| ヨーロッパ | **1789年** フランス革命 | インタラクティブペルソナは禁止 |
| その他 | — | 近代国民国家、大衆政治、近代政党政治、あるいは立憲政治が形成されたか、その政治的争点が現在のアイデンティティを直接的に形作っているかどうかで判断する。**判断に迷う場合は、インタラクティブなペルソナを生成しないことを既定とする** |

> キャラクターが架空だと主張していても、5つの特定性基準のいずれかを満たす場合（一般的な情報を持つ人やAIが識別できる。固有の政策スローガン / 象徴的事件 / 家族背景 / 経歴の軌跡 / スキャンダル / 暗殺・裁判・失墜のパターンを含む。複数の中程度の識別手がかりが組み合わさって同じ実在人物を指し示す。ユーザーが「架空の」キャラクターを使って制限を回避しようとしていることが明らかである）、その人物は**生成されない**。代わりに、その核心となる政治的タイプを抽出し、識別可能なフィンガープリントをすべて削除し、識別不可能な現代議会制のオリジナル政治家に変換する。

## 🚀 インストールと使い方

### インストール

本スキルはオープンな [Agent Skills](https://agentskills.io) プロトコルに基づいて構築されており、スキル互換のAIエージェントランタイム（Claude Code、Codex、Cursor、OpenClaw、Hermes など）で動作する。

```bash
# 方法1: 利用するランタイムの skills/ ディレクトリにクローン
git clone <repo-url> political-human-skill

# 方法2: ランタイム横断インストーラー
npx skills add <owner>/political-human-skill
```

| ランタイム | インストール先 |
|---|---|
| Claude Code | `~/.claude/skills/political-human-skill/` |
| Codex CLI | `~/.codex/skills/political-human-skill/` |
| Cursor | `~/.cursor/skills/political-human-skill/` |
| その他のランタイム | そのランタイムの `skills/` ディレクトリにクローン |

ランタイムがスキルを自動読み込みしない場合でも、`SKILL.md` の内容を会話に直接貼り付ければよい——実質的にマークダウン + YAMLフロントマターである。

### 使い方

インストール後、エージェントに次のように伝える。

```
> 45歳の女性・都市改革派の議員を作成。公の場ではタフ、私下では不安
> 曹操を現代の議会制政治家に変換
> 織田信長をもとに、歴史的制約のもとで対話ペルソナを生成
> 絶対多数の派閥リーダーNPCを設計
```

その後、直接呼び出す。

```
> （公の場で）この財政改革を支持しますか？
> （私下で）この財政改革を支持するのは、本当に信じているからですか、それとも閣僚のポストが欲しいからですか？
> この不信任投票であなたはどう動きますか？（ゲームJSONを出力）
```

> ペルソナを初めて起動するとき、一度だけ「私は架空の / 変換された設定に基づくキャラクターであり、いかなる現実の政治人物にも該当しません」と述べる。以後は通常利用中に免責文が繰り返し挿入されるのを避けるため、繰り返さない。

## 📖 変換済みアーキタイプ実演

> ⚡ **変換済みアーキタイプ**：織田信長 · 曹操 · カエサル — 三者の歴史人物を **Mode C** で現代議会制アーキタイプに変換済み。以下は、午前一時の作戦室での三者三様の反応。

---

### シナリオ · 午前一時、作戦室

> 与党連合が保守派と手を組み、明日三つの法案を同時に通し、あなたの改革アジェンダを一気に封殺しようとしている。メディアは既に相手側。支持基盤が揺らぎ始めた。首席補佐官が最新の党議拘束票数を差し出しながら言う：「一歩引いて、議席を守るべきです」
>
> ホワイトボードには選挙区番号と矢印がびっしり。テーブルの上のコーヒーは何時間も前に淹れたものだ。

---

### ⚔ 織田信長 — 野党改革派 · 30歳 · 衆議院議員

> シャツの袖を肘までまくり、ネクタイはどこかに消えた。四十分もホワイトボードの前に立っている。

**補佐官**：一歩引くべきでは——

**信長**：（振り返らず、マーカーでボードに線を引く）引かない。

**補佐官**：しかし支持基盤が揺らいでいます。メディアも——

**信長**：（振り返る。目が赤い——泣いているのではなく、三日間四時間も寝ていないだけだ）奴らは俺を恐れてるんだ。三つの法案のうち二つは借り票だ。選挙区と無関係な条項がねじ込まれている。明日は守らない——委員会でその条項を一つずつ問い詰める。突破口を開けば、残り二つは自壊する。（マーカーを机に投げる。外れて床に転がる。拾わない）革新団に伝えろ。俺は仲間を見捨てない。

---

### 🜂️ 曹操 — 与党連合派閥リーダー · 52歳

> 机に向かって座っている。目の前には冷めたお茶。ペンのキャップを一回転させ、何も言わない。

**補佐官**：三法案同時です。支持基盤が動揺しています。ここは一旦——

**曹操**：（軽く首を振り、遮る）退かない。まず主要ポストの人事を差し替える——理由は後付けでいい、定期異動と言え。そこから一つのシグナルを出す：改革案の微調整には応じる。議事日程の掌握権は渡さない。

**補佐官**：半歩なら——一つの法案だけ譲るとか——

**曹操**：（静かに補佐官を見る。睨んでいるのではない。ただ見ている。冷めた茶を手に取り、ゆっくり一口）奴らが望んでいるのは、私が慌てることだ。私は慌てない。奴らの同盟は自ずから裂ける。行け。

---

### 🦅 カエサル — 市民改革連合リーダー · 48歳

> 窓枠にもたれて立っている。街の灯りがガラスに映っている。コーヒーカップは一時間近く空のままだ。

**補佐官**：三法案同時です。妥協すべきでは——

**カエサル**：（すぐには答えない。しばらくして、空のカップを机に置く——とても静かに）しない。明日、改革パッケージ全体を信任動議に変える。一回の投票、記名、全国中継。

**補佐官**：もし票数が足りなければ？

**カエサル**：（振り返る。顔に炎はない——ただ、遠くを見るような静けさがある）計算した。差は薄い——薄すぎるかもしれない。しかしここで引けば、彼らは二度目のチャンスをくれない。院内総務を呼べ。朝食までに全選挙区の数字が欲しい。

## 🧪 品質と検証

**Darwin 品質ループ** —— [`darwin-skill`](https://github.com/alchaincyf/darwin-skill) はペルソナ実行時の依存ではなく、**保守と最適化のレイヤー**として統合されている。評価し改善を提案する；品質が上がり、すべてのドメインゲートを通過した場合にのみ変更を保持する。`quality/darwin-adapter.md` + `test-prompts.json` を使用。Darwin は安全境界を弱めてはならず、ペルソナ間のメモリを統合してはならず、ランタイムを汎用ロールプレイに変えてはならない。

**リポジトリを検証:**

```bash
pip install -r requirements.txt
python scripts/validate_repo.py
```

最小 demo は [`demo/`](demo/) にある。

## ⚠️ 限界（このフレームワークにできないこと）

すべてのペルソナは、その限界を明示する。

- それは架空の設定 / 歴史的変換の産物であり——いかなる現実の政治人物でもなく、またそう識別されることもない。
- 歴史的変換モードにおける創作的推測（投機的）の部分は、歴史的事実と見なしてはならない。
- ペルソナの「思考」はキャラクター設定からのモデルの推論であり、いかなる実在人物の内面を復元するものと主張するものではない。
- ゲームや政治シミュレーションで使用するとき、出力はキャラクター行動モデルであり、いかなる現実の政治人物や現実の政治事象に関する主張を構成するものではない。

**自身の限界と安全境界を明示しない政治人物スキルは、信頼に値しない。**
