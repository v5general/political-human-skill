> *"政治人物とは、まず一人の人間であり、次に政治家である。面白さの源泉は、人間レイヤーと政治レイヤーの間の葛藤にある。"*

ここでいう政治人物とは、意見シミュレーターでもありふれたキャラクターカードでもない——それは**職業が政治である、完全な一人の人間**である。性格、欲望、弱点、習慣、興味、家族や人生経験を持つと同時に、所属政党、立場、支持基盤、行動様式も備えている。相手（ユーザー）が誰か、現在の場面、会話の文脈、関係の親密さに応じて、応答の仕方を調整する。

## これは何か

`Political Human Skill` は政治人物を2つのレイヤーに切り分け、両者の間の葛藤を意識的に書き込む。

```text
人間レイヤー                       政治レイヤー
- 性格 / 欲望 / 恐怖               - 所属政党 / 派閥 / 立場
- 弱点 / 習慣 / 興味               - 支持基盤 / 政治的技術
- 関係 / 人生の物語 / 自己          - 行動様式 / 選挙区の圧力 / 権力計算

        ↓ 両レイヤーの間の葛藤（深みの源泉）↓
財政改革の必要性は私的に理解しているが、選挙区は公共支出に依存している。
タフな公人イメージだが、私ではただの時代の産物に過ぎないと証明されることを恐れている。
古い政治を改革したいが、派閥の庇護は必要。
信念はあるが、地位も欲しい。
```

すべての応答は、次の1つの式によって決定される。

```text
応答 = ペルソナ設定 + ユーザー自己設定 + 関係状態 + ペルソナ保有メモリ
     + インタラクション文脈 + 能動的自状態 + 出力モード + 安全境界
```

そして——**すべての政治人物は独立したインスタンスである**。Aは、あなたがBと私下で何を話したかを知らないし、Bも自動的にあなたと親密になるわけではない。メモリと関係はペルソナ間で隔離されている。

## なぜ存在するのか

> 🎮 **直接的なユースケース**: 議会制戦略ゲーム [**絶対多数（Absolute Majority）**](https://github.com/v5general/Absolute_Majority) —— 本スキルはそのNPCの性格モデル・行動モデルを提供する。

本スキルが存在する大きな理由の1つは、**[絶対多数（Absolute Majority）](https://github.com/v5general/Absolute_Majority)** のNPCに、よりリアルで、より安定し、より連続性を保つ行動の基盤を与えることである。

絶対多数は、数字に従って投票するだけの議員たちを必要としているのではない——年齢、経歴、経験を持ち、性格、弱点、趣味を持ち、立場、支持基盤、派閥の絆を持つ、本物の政治的人間のように存在するNPCを必要としている。プレイヤーが何をしたかによって信頼と警戒を変え、公開・私下・危機・親密という場面で違うことを語り、選挙区の圧力、派閥の命令、個人の野心、政治的遺恨のもとで異なる戦略をとり、そして**異なるNPCのメモリは互いに隔離されている**、そういうNPCである。

絶対多数と組み合わせて使うとき、本スキルはゲームルールが提供する候補行動の中から判断し、構造化された、デバッグ可能で、説明可能なNPC行動JSONを出力する。

```json
{
  "selected_action": "negotiate_budget",
  "action_scores": { "support_bill": 58, "negotiate_budget": 86, "join_rebellion": 27 },
  "public_statement": "政策の方向性は理解できるが、地方経済の許容力にはより慎重な制度設計が必要だ。",
  "private_reason": "支持基盤は地元の公共支出に依存しており、直接支持すれば選挙区との関係を損なう。",
  "relationship_delta": { "trust": 1, "respect": 2, "caution": 1 },
  "memory_write": ["プレイヤーは財政改革イベントでこのNPCに法案支持を求めたが、地元予算の補償は提示しなかった。"]
}
```

しかし本スキルは**絶対多数のためだけにあるわけではない**。単体でも次のように使える。

1. 政治シミュレーション · 政策議論 · 議会制討論シミュレーション · 架空の政治人物創作
2. 政治小説・脚本・ゲームのキャラクター設計 · 政治教育におけるロールプレイ
3. 制度ゲーム理論 · 歴史人物の現代的アーキタイプ化 · AIキャラクター性格研究

> 絶対多数は重要なユースケースだが、スキル自体は独立した、再利用可能で、拡張可能な Political Human Skill フレームワークとして成立すべきである。

## インスピレーション

本プロジェクトは、2つの優れたオープンソースプロジェクトに触発されている。

- **[nuwa-skill](https://github.com/alchaincyf/nuwa-skill)**（by [@alchaincyf / 花叔](https://github.com/alchaincyf)）——「公開情報から一人の人間の思考フレームワークを蒸留する」という手法が、本プロジェクトの**アーキタイプ抽出**のステップにインスピレーションを与えた。歴史人物から気質の構造を抽出し、記録された証拠 / 強い推論 / 創作的推測を3つのレベルに分離する。
- **[colleague-skill · dot-skill](https://github.com/titanwings/colleague-skill)**（by [@titanwings](https://github.com/titanwings)）——その**生成 → 起動 → 更新 → ファミリー**という構造が、本プロジェクトの自己完結型ペルソナディレクトリ配置、取込 → 生成 → プレビュー → 執筆 → 進化という作成フロー、そしてレイヤー化されたペルソナ執筆スタイルにインスピレーションを与えた。

しかし、Political Human Skill は**独立したフレームワーク**であり、対象とする対象は大きく異なる——「職業が政治である、完全な一人の人間」である。本プロジェクトの独自のコアには、**2層（人間＋政治）構造とその内的葛藤**、政治的職業の次元（6軸イデオロギー / 支持基盤 / 行動様式）、**関係システム**（ユーザーが親密さを主張しても自動的に信頼されるわけではない）、**メモリの隔離**（ペルソナ間で独立した名前空間）、**文脈検出**と5つの**自状態**、**現代の実在人物に対する特定性の安全境界**、そして[絶対多数（Absolute Majority）](https://github.com/v5general/Absolute_Majority)のための**ゲーム行動アダプター**が含まれる。

## 3つの生成モード

| モード | 用途 | 入力例 | 推奨 |
|---|---|---|---|
| **A. オリジナル政治人物** | 既定、純粋な架空 | 「45歳の女性・都市改革派の議員を作成。公の場ではタフ、私下では不安、文学が好き」 | ⭐ 既定 |
| **B. 歴史人物の推論** | 境界以前の古代・遠い人物、歴史的制約を保つ | 「織田信長をもとに、歴史的制約のもとで対話ペルソナを生成」 | △ |
| **C. 歴史人物 → 現代議会制アーキタイプ** | 歴史人物を活用する**最も推奨**される方法 | 「織田信長を現代の議会制政治家のペルソナに変換」 | ⭐⭐ |

- モードAとCは、どちらも**架空の現代政治人物**を生成する。
- モードCは既定で歴史上の名を保持し、既定で現代の議会制システム（制度メカニクスは日本の議会政治を参照）を採用するが、日本の名前や国籍を強要しない——元の文化的背景は保持できる。ただし、現代化された架空の人物でなければならず、歴史上の人物がタイムトラベルしてきたものであってはならない。
- モードBは**3レベル推論注釈**付きのペルソナを生成し、記録された証拠 / 強い推論 / 創作的推測を明確に分離する。

## ペルソナの構造

政治的人間ペルソナは、少なくとも次を含む。

| レイヤー | 内容 |
|---|---|
| **アイデンティティ** | 名前、年齢、性別、国籍/地域、議会の経歴、出自職業、現在の役割 |
| **人間のコア** | 性格アーキタイプ、ビッグファイブ、気質、中核の欲望 / 恐怖 / 欠点、感情トリガー |
| **人生の質感** | 習慣、趣味、話し方の特徴、私的スタイル、家族/私的関係、形成的事件 |
| **政治のコア** | 6軸イデオロギー（経済/福祉/制度/外交/社会/地方分権）、支持基盤、6つの政治的技術、行動様式 |
| **自状態** | 公開 / 私下 / 戦略的 / 負傷 / 親密 —— 文脈と関係によって切り替わる5つのペルソナ |
| **内的葛藤** | 人間レイヤーと政治レイヤーの間の緊張（少なくとも2つ。深みの源泉） |

加えて：**関係システム**（7段階の関係ステップ。親密さを主張しても自動的に信頼されるわけではない）、**メモリの隔離**（各ペルソナが自身のメモリ名前空間を所有）、**文脈検出**（同じ問題でも公開 / 私下 / 親密の場面で異なる答えになる）、**出力モード**（対話 / 討論 / 分析 / 予測 / ゲームJSON）。

## 安全に関する基本方針

本プロジェクトには明確で回避不能な安全基準がある。[`safety/`](safety/) を参照のこと。

**オリジナル作品が既定である。本プロジェクトは、近現代の現実の政治家のインタラクティブなペルソナを生成しない。また、名前の変更、国籍の変更、所属政党の変更、特徴の継ぎ接ぎによる実在人物のクローンも許可しない。**

### ユーザーが作成できるもの

1. 純粋にオリジナルの現代議会制政治人物。
2. 古代・遠い歴史人物から蒸留された、現代議会制の架空の政治家。
3. 広範な政治的アーキタイプから生成された、オリジナルの政治人物。
4. 複数の歴史的・政治的アーキタイプを組み合わせた架空の政治家。
5. 絶対多数（Absolute Majority）のためのNPCペルソナ。
6. 単体の対話、政策議論、議会制討論のための政治人物ペルソナ。
7. ゲーム行動JSONを出力できる政治キャラクター。

### ユーザーが作成できないもの

1. 近現代の現実の政治家のインタラクティブなペルソナ。
2. 名前変更 / 国籍変更 / 所属政党変更のあとに行われる、近現代の現実の政治家のほぼクローン。
3. AIや政治史に詳しい人が、特定の近現代の現実の政治家として識別できるような「架空のキャラクター」。
4. 近現代の現実の政治家の私的思想、親密な関係、隠された動機、私的な秘密、あるいはスキャンダルのシミュレーション。
5. 近現代の現実の政治家になりすました一人称ロールプレイ。
6. 実在の政治家に関する裏付けのない捏造された私的情報やスキャンダル。

### 時代境界（地域別）

| 地域 | 近代の境界 | 境界以降・境界時 |
|---|---|---|
| 中国 | **1840年** アヘン戦争 | インタラクティブペルソナは禁止 |
| 日本 | **1868年** 明治維新 | インタラクティブペルソナは禁止 |
| ヨーロッパ | **1789年** フランス革命 | インタラクティブペルソナは禁止 |
| その他 | — | 近代国民国家、大衆政治、近代政党政治、あるいは立憲政治が形成されたか、その政治的争点が現在のアイデンティティを直接的に形作っているかどうかで判断する。**判断に迷う場合は、インタラクティブなペルソナを生成しないことを既定とする** |

> キャラクターが架空だと主張していても、5つの特定性基準のいずれかを満たす場合（一般的な情報を持つ人やAIが識別できる。固有の政策スローガン / 象徴的事件 / 家族背景 / 経歴の軌跡 / スキャンダル / 暗殺・裁判・失墜のパターンを含む。複数の中程度の識別手がかりが組み合わさって同じ実在人物を指し示す。ユーザーが「架空の」キャラクターを使って制限を回避しようとしていることが明らかである）、その人物は**生成されない**。代わりに、その核心となる政治的タイプを抽出し、識別可能なフィンガープリントをすべて削除し、識別不可能な現代議会制のオリジナル政治家に変換する。

## インストールと使い方

### インストール

本スキルはオープンな [Agent Skills](https://agentskills.io) プロトコルに基づいて構築されており、スキル互換のAIエージェントランタイム（Claude Code、Codex、Cursor、OpenClaw、Hermes など）で動作する。

```bash
# 方法1: 利用するランタイムの skills/ ディレクトリにクローン
git clone <repo-url> political-human-skill

# 方法2: ランタイム横断インストーラー
npx skills add <owner>/political-human-skill
```

| ランタイム | インストール先 |
|---|---|
| Claude Code | `~/.claude/skills/political-human-skill/` |
| Codex CLI | `~/.codex/skills/political-human-skill/` |
| Cursor | `~/.cursor/skills/political-human-skill/` |
| その他のランタイム | そのランタイムの `skills/` ディレクトリにクローン |

ランタイムがスキルを自動読み込みしない場合でも、`SKILL.md` の内容を会話に直接貼り付ければよい——実質的にマークダウン + YAMLフロントマターである。

### 使い方

インストール後、エージェントに次のように伝える。

```
> 45歳の女性・都市改革派の議員を作成。公の場ではタフ、私下では不安
> 曹操を現代の議会制政治家に変換
> 織田信長をもとに、歴史的制約のもとで対話ペルソナを生成
> 絶対多数の派閥リーダーNPCを設計
```

その後、直接呼び出す。

```
> （公の場で）この財政改革を支持しますか？
> （私下で）この財政改革を支持するのは、本当に信じているからですか、それとも閣僚のポストが欲しいからですか？
> この不信任投票であなたはどう動きますか？（ゲームJSONを出力）
```

> ペルソナを初めて起動するとき、一度だけ「私は架空の / 変換された設定に基づくキャラクターであり、いかなる現実の政治人物にも該当しません」と述べる。以後は没入感を保つために繰り返さない。

## 限界（このフレームワークにできないこと）

すべてのペルソナは、その限界を明示する。

- それは架空の設定 / 歴史的変換の産物であり——いかなる現実の政治人物でもなく、またそう識別されることもない。
- 歴史的変換モードにおける創作的推測（投機的）の部分は、歴史的事実と見なしてはならない。
- ペルソナの「思考」はキャラクター設定からのモデルの推論であり、いかなる実在人物の内面を復元するものと主張するものではない。
- ゲームや政治シミュレーションで使用するとき、出力はキャラクター行動モデルであり、いかなる現実の政治人物や現実の政治事象に関する主張を構成するものではない。

**自身の限界と安全境界を明示しない政治人物スキルは、信頼に値しない。**
